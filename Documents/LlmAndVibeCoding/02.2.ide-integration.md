# 2.2 전환점: IDE 통합의 시작 (2023-2024)

---

## 1세대: IDE 통합의 시작 (2023-2024)

**개발 환경과의 융합**
- IDE에 AI를 직접 통합하는 시도

**대표 툴**

**GitHub Copilot** (2023년 초)
- Microsoft와 OpenAI 협력
- VSCode, JetBrains IDE 지원
- 인라인 코드 자동완성

---

## 1세대: 대표 툴 (계속)

**Cursor** (2023년 중반)
- VSCode 포크 기반 AI 네이티브 IDE
- GPT-4 기반 코드 생성
- 프로젝트 전체 컨텍스트 이해

**Cline** (2023년 후반)
- VSCode 확장 형태
- 다양한 LLM 모델 선택 가능
- 채팅 기반 코드 수정

---

## 1세대: 핵심 특징

**IDE 네이티브 통합**
- 코드 에디터 내에서 직접 AI 활용
- 실시간 코드 제안 및 자동완성
- 파일 탐색기와 연동

**프로젝트 컨텍스트 인식**
- 현재 작업 중인 파일 내용 이해
- 프로젝트 구조 파악 (제한적)
- 관련 파일 참조 가능

---

## 1세대: 개선점

| 항목 | 0세대 (채팅) | 1세대 (IDE) |
|------|-------------|------------|
| 작업 환경 | 브라우저 ↔ IDE | IDE 통합 |
| 컨텍스트 | 대화 내용만 | 프로젝트 파일 |
| 워크플로우 | 복사-붙여넣기 | 직접 편집 |
| 생산성 | 낮음 | 중간 |

---

## 1세대: 한계

**여전한 제약**
- 단일 파일 중심 작업 (멀티파일 제한적)
- 수동 개입 필요 (파일 생성/삭제)
- 로컬 리소스 의존 (메모리 사용 증가)
