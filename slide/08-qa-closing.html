8. 자주 묻는 질문 (Q&A)

도구 선택 관련

Q1: 처음 시작하는 개발자에게 어떤 도구를 추천하나요?

추천: 1세대 (Cursor)

이유:

  • GUI 환경이 직관적하고 학습 곡선이 낮음
  • 시각적 피드백으로 변경사항을 즉시 확인 가능
  • VSCode와 유사한 인터페이스로 친숙함

Q1: 학습 경로

Week 1-2: Cursor 기본 기능 익히기

  • Cmd/Ctrl + K로 간단한 요청
  • diff 모드로 변경사항 확인
  • 작은 컴포넌트 생성 연습

Week 3-4: 실전 프로젝트 적용

  • 사이드 프로젝트에 활용
  • 코드 리뷰 습관 형성
  • 베스트 프랙티스 학습

Q2: 권장사항

두 도구를 모두 사용하는 것도 좋은 전략

프론트엔드 작업: Cursor (UI 프리뷰 필요)
백엔드/인프라: Claude Code (자동화 효율적)
테스트/배포: Claude Code (스크립트 자동화)

Q3: 무료로 시작할 수 있는 도구

무료 옵션

Cline (VSCode 확장)

  • 완전 무료 오픈소스
  • 다양한 LLM 모델 선택 가능
  • 자체 API 키 사용 시 비용만 지불

ChatGPT 무료 버전

  • 기본적인 코드 생성 가능
  • 복사-붙여넣기 방식
  • 제한적이지만 학습용으로 적합

Q3: 추천 전략

1단계: ChatGPT 무료 버전으로 개념 이해
2단계: Cline으로 IDE 통합 경험
3단계: 생산성 확인 후 유료 도구 선택

비용 및 ROI 관련

Q4: 월 $20가 비싸지 않나요?

ROI 분석 (중급 개발자 기준)

시간 절감: 주 15시간 (월 60시간)
개발자 시급: $50 (평균)
절감 비용: 60시간 × $50 = $3,000/월

도구 비용: $20/월
순이익: $2,980/월

ROI: 14,900%

Q5: 회사 단체 도입 시 비용 절감

라이선스 협상 전략

소규모 팀 (5-10명)

  • 개별 구독으로 시작
  • 성과 측정 후 협상
  • 예상 비용: $100-200/월

중형 팀 (10-20명)

  • 단체 할인 문의 (10-15% 할인)
  • 연간 계약으로 추가 할인
  • 예상 비용: $200-350/월

Q5: 협상 팁

Q6: 기업용 솔루션

보안 강화 옵션

  • Cursor Business: 데이터 보관 없음
  • Claude Code Enterprise: SOC 2 인증, 데이터 격리

Q7: AI 생성 코드의 저작권

저작권 현황 (2024년 기준)

대부분의 도구 정책:

  • 사용자가 생성된 코드의 소유권을 가짐
  • AI는 도구일 뿐, 저작자가 아님
  • 상업적 사용 가능

주의사항:

⚠️ 확인 필요:
- 각 도구의 이용 약관 확인
- 오픈소스 라이선스 준수
- 제3자 코드 사용 여부

학습 및 실무 적용

Q8: AI 도구 사용으로 개발 실력이 떨어지지 않을까요?

올바른 사용 (실력 향상):

✅ AI 코드를 학습 자료로 활용
✅ 생성된 코드를 분석하고 이해
✅ 다양한 구현 방법 비교
✅ 베스트 프랙티스 학습

잘못된 사용 (실력 저하 위험):

❌ AI에게 모든 것을 맡김
❌ 생성된 코드를 이해하지 않고 사용
❌ 기본 개념 학습 생략
❌ 디버깅 능력 상실

더 효율적인 알고리즘 사용해줘."

Q15: 성능/속도 문제

느린 응답 시간 해결:

원인:
- 프로젝트 크기가 매우 큼
- 너무 많은 파일 참조
- 복잡한 요청

해결:
✅ 요청을 작은 단위로 분할
✅ 관련 파일만 컨텍스트에 포함
✅ 점진적 작업 진행
